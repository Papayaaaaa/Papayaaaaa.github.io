一个假程序员的追星惨案

我再也不要随便立flag玩python了QAQ
起因时很久很久之前要给龙龙投一个投票，我特别没耐心so基本重复超过10次的工作基本上我就会开始上网搜代码了orz，后来发现百度盖楼对数据帮助比较大，所以在网上找（抄）了一点自动回帖代码试了下，主要参考的的代码是 魔力Python » 小工具：百度贴吧自动顶贴机不过没有用上cookie函数（我不知道是不是我的打开方式不对我用cookie登录买的小号各种不成功，理论上是可以的well一定是我这条咸鱼的问题）
主要使用的工具包是selenium包，可以使用pip安装pip install -U selenium
PS.需要下载与电脑Chorme的版本相对应的chormedriver，版本一定要对应！！
然后把chormedriver复制到至谷歌浏览器目录（如 C:\Program Files\Google\Chrome\Application）以及当前运行的python根目录（例如我的是D:\Anaconda3）如果不行就把谷歌浏览器目录添加到环境变量path
这里主要参考的是博客 解决chromedriver executable needs to be in PATH问题

代码下载

url.txt——需要盖楼的链接，可以每天修改，现在里面是4月10号的朱一龙贴吧数据楼
reply.txt—回复，每次回复取一条并添加随机字母后缀防止吞楼
tieba.py—python脚本，python版本是3.6

代码改到最后还是用了人工登录，看到有人说这叫人工干预爬虫hhhhhhh，但是确实我买的小号都不好用，人工登录可能还要方便一点，函数login()里面是自动登录首页的代码，但是我后面登录次数太多了需要验证就不太好用了所以主函数里面写的还是需要人工登录一下，这里只放主函数

人工登录时间间隔我设置的是120秒（因为太快了有时候来不及接收验证码），回复时间间隔设置的是300秒，没有试过更快的，怕被封号orz，我基本上不会出现回复需要滑动验证的情况，如果出现了可以手动帮忙验证下，我懒得继续写滑动验证码了，隔一段时间看一眼就行
——————————————————————————
如果从来没有接触过python的话，可以参考这个网站安装一个python，然后打开python自带的编译器IDLE(Python GUI)，从IDLE–>File–>Open打开python脚本，然后选择菜单Run–>Run Module(当然也可以打开python脚本之后直接F5)

PS. tb上面像又自动顶帖机软件卖。。。可以直接买那个，也不是特别贵，我还没买过但是我感觉别人那个做的比我这个强多了

最后祝大家快乐追星~

Github/Gitee+hexo搭建博客

  1. 注册Github/Gitee账号，安装Git客户端，生成并配置GitHub公钥，截图步骤略（这一步是为了让后面推送时不需要每次都输入账号密码）
  2. 创建仓库并在设置中开启Git Pages功能
    最好两者仓库的名字都相同，Github Page要求仓库名字为 [账号名].github.io，Gitee无要求但不可以以.git结尾
    例如我的github帐号名为Papayaaaaa
    则Github Page 仓库为名为 Papayaaaaa.github.io Gitee仓库名为Papayaaaaa
    Github(Pages配置在settings页面最底部，默认配置就ok) Gitee
  3. 安装Node.js，网址略，安装完成后查看nodejs版本及npm版本验证安装是否完成
  4. 安装Hexo，在电脑中创建一个新文件夹，命名为Papayaaaaa.github.io作为博客文件夹
    看到有博客说名称和仓库名不同会有问题但是理论上不同名也应该ok的，不想麻烦的话建议还是同名
  5. 在文件夹中打开cmd（按住Shift右键）
    使用npm命令安装Hexo
    npm install -g hexo-cli
  6. 安装Git插件 cmd输入
    npm install hexo-deployer-git --save
  7. 初始化博客，cmd输入
    hexo init blog
  8. 创建一个新页面， cmd输入
    hexo new "My first blog"
    这句命令即创建一个新的名为My first blog的页面，页面以.md文件储存在/source/_post/文件夹下，页面语言为Markdown
  9. 生成静态博客页面 hexo g g即generate，生成博客
  10. 本地测试博客框架 hexo s s即server， 可以在localhost:4000查看本地博客框架
  11. 将本地博客推送到Github/Gitee
    修改站点根目录下的站点配置文件_config.yml，记事本或Notepad打开
    修改最下面一行 文件最开头可以设置博客标题等 修改完保存文件，本地博客根目录下命令行
    hexo clean
    hexo g
    hexo d##即deploy，进行远程推送
  12. 每次自己在本地写好博客都可以通过这三个命令进行页面生成与推送
  13. 貌似推送到gitee需要在source文件夹下建立一个Staticfile文件否则无法识别静态网页（？不知道为啥）

遇到的一些问题：
之前在Gitee上的仓库名称和我账号名称不一样，推送之后在Gitee上就无法显示出页面，设置_config.yml的url和root属性可以解决但设置之后Github上又显示不出来框架只显示静态页面，搞了半天也不知都为啥，最后只能删了重来大家都统一命名，猜测是命名不同会导致路径的一些变化，dbq本人真的是纯正的半桶水orz

PS：
码云（Gitee）上推送后无法自动部署网站，每次推送后需要自己手动部署（自动部署需要购买VIP 99r/year），其实用Gitee本来是觉得会快一点但是好像也没有快很多的样子orz.

PPS：
绑定域名就按网上来就ok，但是注意直接在仓库根目录建立CNAME文件在第二次推送的时候会被覆盖掉因此需要在本地目录/source文件夹下建立CNAME文件。
