Node.js部署静态网页

发表于 2019-07-07

Node.js真的是hin牛逼，基本上是傻瓜式操作了，而且比tomcat什么的要方便多了
这个的起源是准备给明天过生日的的基友做一个生日快乐的网站，奈何自己的前端水平着实有点烂，就是属于那种完全扶不上墙的烂，所以准备github上面抄一个代码，搞简单一点好了

  1. 安装node.js

  2. 创建网页目录

  3. 在目录下写一个server.js，这里监听的端口是9999，大家也可以换成自己的

这里遇到的一个问题是，开启了端口也无法访问，原因是server.listen(9999)这个函数第二个参数指定了本地localhost地址，于是只能本机访问了。

  1. 防火墙打开9999端口

  2. 放上自己的静态网页index.js

  3. 在对应目录下运行node server

  4. 这种运行方式人走了就没办法继续访问了，因此需要下载pm2工具管理node服务

  5. pm2开启服务并监控

  6. pm2关闭服务

Springboot+MyBatis+MySQL

发表于 2019-06-27

轮到自己写博客就知道那种手把手事无巨细的博客有多么贴心了，自己写就很难搞，起因是之前一个课设准备做空气质量的，但是下载的空气质量数据是每天一个文档记录全国数据的使用不是很方便，就准备存到数据库里面这样，本来还异想天开准备一个站点一个表的后来发现原数据问题很多还是存成了一个表比较合适（不过其实理论上多个表更适合查询吧？）
具体代码就不贴了，不想暴露自己菜的扣脚的coding水平，大概就是Springboot+Mybatis连MySQL的demo水平，稍微记录下自己遇到的一些问题吧

AQIprocess

  • 这边有遇到一个神奇的时区问题，好像是高版本的MySQL会出现的一个问题，说程序的时区和MySQL的时区不一致大概这样，然后解决方法非常简单，在配置文件里面加上时区约束就可以了
  • Mapper.xml无法识别的问题、也是在配置文件里面加上mybatis.mapper-locations=……参数
  • 因为不想调试的时候重复删表，同时源数据里面有重复的记录，所以这边把SQL语句里面的insert into改为replace into（仅适用于MySQL，我爱MySQL）这样可以在逐渐重复的时候替换掉原有数据，如果是别的数据库的话可以在insert语句后面加上select条件
  • 因为数据记录比较多（最后貌似是1450263条记录）所以这边开了一个springboot自带的线程池跑，但是因为文件太多了所以会有线程队列满了而无可用线程的问题，并且线程跑的时间太长有被主线程退出的风险，因此需要用到CountDownLatch类用来阻塞主线程，使主线程等待直到所有的线程跑完这样
  • 判断字符串为空的问题，如果需要判断str是不是空串””需要用到str.IsEmpty,直接==比较容易出问题，这个问题其实本科就遇到过了但是太久没写代码了又都还回去了

Github/Gitee+hexo搭建博客

发表于 2019-03-29
  1. 注册Github/Gitee账号，安装Git客户端，生成并配置GitHub公钥，截图步骤略（这一步是为了让后面推送时不需要每次都输入账号密码）
  2. 创建仓库并在设置中开启Git Pages功能
    最好两者仓库的名字都相同，Github Page要求仓库名字为 [账号名].github.io，Gitee无要求但不可以以.git结尾
    例如我的github帐号名为Papayaaaaa
    则Github Page 仓库为名为 Papayaaaaa.github.io Gitee仓库名为Papayaaaaa
    Github(Pages配置在settings页面最底部，默认配置就ok) Gitee
  3. 安装Node.js，网址略，安装完成后查看nodejs版本及npm版本验证安装是否完成
  4. 安装Hexo，在电脑中创建一个新文件夹，命名为Papayaaaaa.github.io作为博客文件夹
    看到有博客说名称和仓库名不同会有问题但是理论上不同名也应该ok的，不想麻烦的话建议还是同名
  5. 在文件夹中打开cmd（按住Shift右键）
    使用npm命令安装Hexo
    npm install -g hexo-cli
  6. 安装Git插件 cmd输入
    npm install hexo-deployer-git --save
  7. 初始化博客，cmd输入
    hexo init blog
  8. 创建一个新页面， cmd输入
    hexo new "My first blog"
    这句命令即创建一个新的名为My first blog的页面，页面以.md文件储存在/source/_post/文件夹下，页面语言为Markdown
  9. 生成静态博客页面 hexo g g即generate，生成博客
  10. 本地测试博客框架 hexo s s即server， 可以在localhost:4000查看本地博客框架
  11. 将本地博客推送到Github/Gitee
    修改站点根目录下的站点配置文件_config.yml，记事本或Notepad打开
    修改最下面一行 文件最开头可以设置博客标题等 修改完保存文件，本地博客根目录下命令行
    hexo clean
    hexo g
    hexo d##即deploy，进行远程推送
  12. 每次自己在本地写好博客都可以通过这三个命令进行页面生成与推送
  13. 貌似推送到gitee需要在source文件夹下建立一个Staticfile文件否则无法识别静态网页（？不知道为啥）

遇到的一些问题：
之前在Gitee上的仓库名称和我账号名称不一样，推送之后在Gitee上就无法显示出页面，设置_config.yml的url和root属性可以解决但设置之后Github上又显示不出来框架只显示静态页面，搞了半天也不知都为啥，最后只能删了重来大家都统一命名，猜测是命名不同会导致路径的一些变化，dbq本人真的是纯正的半桶水orz

PS：
码云（Gitee）上推送后无法自动部署网站，每次推送后需要自己手动部署（自动部署需要购买VIP 99r/year），其实用Gitee本来是觉得会快一点但是好像也没有快很多的样子orz.

PPS：
绑定域名就按网上来就ok，但是注意直接在仓库根目录建立CNAME文件在第二次推送的时候会被覆盖掉因此需要在本地目录/source文件夹下建立CNAME文件。
